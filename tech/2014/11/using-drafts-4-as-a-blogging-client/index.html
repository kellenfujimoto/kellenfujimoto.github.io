Using Drafts 4 As A Blogging Client

Wed, Nov 19, 2014 ~300 words

When Agile Tortoise’s awesome update of Drafts for iOS came out with iOS 8, Greg Pierce added a potent extension mechanism into the app. This — paired with a potent directory — opened up a limitless possibility for customization by users, provided they knew what they wanted and how to assemble it. This was a brilliant feature to add, given the geeky slant of Drafts’ user base.

I decided to use this newfound capability to build a lightweight blogging client, feeding posts into an installation of Hugo on my desktop. While I could manually create posts using my iOS text editor of choice and push them to Dropbox, I’d prefer a more automated approach.

The built-in Dropbox share action has most of what I need. I can set up a template which takes care of the post boilerplate for me (publication date, title, etc.) using built in template tags. The only missing pieces were with file names (Hugo uses post filenames as slugs) and declaring a source link for link list posts.

I used a quick regular expression on the title, adding a slug attribute to the draft object, and used line selection for crude link declaration (currently, if I accidentally omit a valid hyperlink in the second line of a link post, there’ll be a very bizarre dead href in the post title). Quick and easy.

Once I have the file saved to Dropbox, my iMac takes over once the file is synced (usually takes a second or three)…
